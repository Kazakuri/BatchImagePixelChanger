BatchImagePixelChanger

Allows the evasion of duplicate image errors

View project onGitHub

About the software.

BatchImagePixelChanger allows a user to take any given image and make multiple copies of the image that are different enough to evade most duplicate image checks.
The algoritm loads in a image and creates copies that picks
image.width() * image.height() * 0.05 pixels (5% of the pixels) randomly from the image (may pick the same pixel twice) and subtracts 5 from the RGB values before saving it to a new file.

Usage.

Using BatchImagePixelChanger is fairly simple, calling the program from the command line you can direct it like so: BatchImagePixelChanger "file-name" copy-count "naming-scheme"
It is recommended to use quotes in the naming scheme and file names so that the program recognizes the spaces in the names.
The program will output files in the format of naming-scheme #.fileext
